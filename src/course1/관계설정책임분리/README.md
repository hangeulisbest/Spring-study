## 원칙과 패턴
1. 개방 폐쇄 원칙 (Open-Close Principle)
```
클래스나 모듈은 확장에는 열려있어야 하고 변경에는 닫혀있어야 한다.
UserDao 는 DB연결방법이라는 기능을 확장하는 데는 열려있다. 
왜냐하면 UserDao에 전혀 영향을 주지 않고 얼마든지 DB연결방법을 바꿀 수 있다.
 또한 UserDao 의 핵심 코드에는 변화가 없다. 따라서 변경에 닫혀있다.
```

2. 전략패턴 
```
전략패턴은 디자인 패턴의 꽃이라고 불린다. 
개방 폐쇄 원칙의 실현에도 가장 잘 들어 맞는 패턴이다.
전략패턴은 자신의 기능 맥락(Context)에서 필요에 따라 변경이 필요한 알고리즘을 
인터페이스를 통해 통째로 외부와 분리를 시킨다. 

UserDao 는 전략패턴의 Context 이다.
Context의 기능중 DB연결방식이라는 알고리즘을 ConnectionMaker 인터페이스로 정의하여
이를 구현한 클래스(=전략)을 바꿔가면서 사용할 수 있게 분리 했다.
```
